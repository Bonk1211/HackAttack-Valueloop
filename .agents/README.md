# Repository Agent Extensions

Active skills in `skills/` are limited to frontend implementation, visual design, image-to-code, and redesign workflows. Use the smallest relevant set for each task; dashboard work should prefer `redesign-existing-projects` and a compatible visual direction such as `minimalist-ui`.

General engineering and planning skills were moved to `archived-skills/non-frontend/`. They are retained for recovery but are not part of the active repository skill set. Move a skill back into `skills/` only when the repository needs that workflow again.

Other extension folders:

- `subagents/` — custom agent definitions
- `memories/` — durable project notes
- `plugins/` — repository-local plugin material
